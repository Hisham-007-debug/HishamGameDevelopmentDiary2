Week 3

The Big Idea

The Team Meeting

Our team of 7 met up to figure out what kind of game we actually wanted to make. After some back and forth, we all agreed on a 2D fighting game. We want it to feel and play a bit like Street Fighter 3.

Finding Our Way

We weren't sure which game engine to use or what art should look like. To get some professional advice, we went to the game modeling lab and talked to Professor Gavin. He gave us two really helpful tips:

  • Use Unity: He said it's very beginner-friendly for a team like ours.
  • Use Pixel Art: This ensures the game runs smoothly on everyone's laptops without lagging.
Week 4

Planning and Roles

Setting the Theme

We had two meetings this week. In the first one, we brainstormed the story and characters. We decided to have 4 playable characters based on elements: Fire, Ice, Earth, and Air. We also planned to have 4 different arenas (stages) that match those elements. We also grew our team to 8 members now that Annie has joined us.

Assigning Jobs

In our second meeting, our producer Aaron gave everyone a specific role so we could start working:

  • Programmers: Me, Aaron, and Niketa.
  • Artists: Annie and two others are handling the characters and backgrounds.
  • UI/GUI: Shayan designs the menus and health bars.
  • Sound: James is making the music and elemental sound effects.

Aaron then created a discord server in which he invited all of us to join. This was a great way to keep in touch and share our progress. He as well created seperate channels for programming and artists so it was easier to manage and not to confuse everyone.

Week 5

Learning the Engine

Independent Work

We didn't have a formal meeting this week because Aaron gave us all homework. The artists started sketching the characters, and my job was to learn Unity and figure out how to get art into the game. I found many helpful YouTube tutorials:

  • Unity Sprite Animation Tutorial
  • 2D Character Movement in Unity

These tutorials taught me how to import sprites from a sprite sheet made from Aseprite or LibreSprite, then add them into Unity.

The Learning Curve

I downloaded a free "Fire Warrior" character from itch.io to practice with. I spent a lot of time watching YouTube tutorials to figure out the technical side. It was a bit tricky at first but then I learned how to:

  1. Splice the sprite sheet: Cutting one big image into separate animation frames.

    Splicing is built into Unity, it is actually seamless, all you have to do is click sprite editor then you add sprite component to the game object, and it will automatically splice it for you.

  2. The Animator: Setting up the logic, so the game knows when to switch from "walking" to "attacking".

    The splicing helps with the animator, because splicing separates each frame for the animation. So for walking and attacking, there were frames made, I had to drag these frames to create the animation clip in Unity.

  3. Coding: I used C# to make the character move when you press a key. Since I already know Java from my other classes, the coding part wasn't too hard—C# is actually pretty similar.

    From the tutorials, I learnt the Legacy Unity Input System, basically hard coding all the movements for the characters. The system knows by default the key bindings for controller and keyboard/mouse, so you must manually just add the control you need to the if statement.

Week 6

Learning Something Horrible

Since the game we are making requires local multiplayer—obviously it needs it, it's a fighting game—two players with two controllers should select their element characters, select the arena, and battle it out. However, this is apparently impossible in Legacy Unity Input System, since it is more difficult to implement local multiplayer.

Unity actually created the new input system for this exact reason, so I searched up tutorials on the new input system and got to work.

Learning the New Input System with Aaron

Aaron, Shayyan, James and I had a meeting for the games project. Aaron and I worked on the new input system. We learnt it's actually easier than the legacy version, because Unity does all the hard coding for us. We basically just have to code in the velocity speed for the movement speed for the characters. Currently we have only draft versoins of the Air element and Earth element characters not final versions of the animations. So, we worked on their movement and attacks.

Week 11 and Week 12

Bringing the Fire Character to Life

Overview

Me, Aaron, and Shayan all worked together during these two weeks to push the game forward. We split up the work and each tackled different parts of the project.

Adding the Fire Character

Darragh made the Fire character animations and I got the job of importing them into Unity with all of their assets. Once I had everything in, I grabbed the player scripts that we had already made for the Air character and gave them a try with the Fire character. To our surprise, they worked perfectly right away! This made us realize something really cool: we could probably use the same script for all the characters. That's going to save us a ton of time going forward.

Fixing Menus and Character Selection

Shayan was busy improving the menus to make them actually work properly. He also added scripts to let players choose their characters and pick which arena they wanted to fight in before jumping into the fight scene. His work made the whole flow of the game feel much smoother.

Crushing Bugs and Building Systems

Aaron had a lot on his plate. There were tons of bugs that needed fixing, mostly because of issues with GitHub that kept messing up our code. He spent most of his time hunting down and squashing those bugs so the game would run cleanly. On top of that, Aaron also built out the text and rounds system for when the game finishes. We're doing best of 3 rounds, so he set that all up so players would see the round count and know when the match was over.

By the end of these two weeks, we had a much more complete game. The Fire character was in, the menus worked, and the whole experience felt way more polished. We were really happy with how everything came together.
